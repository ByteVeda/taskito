taskito uses a pluggable serializer for task arguments and results. The default
is <SdkSwap python={<code>SmartSerializer</code>} node={<code>JsonSerializer</code>} java={<code>JsonSerializer</code>} />,
which <SdkSwap python="uses MessagePack with a cloudpickle fallback for lambdas, closures, and complex objects" node="emits human-readable JSON bytes" java="emits human-readable JSON bytes via Jackson" />.

<SdkOnly sdk="python">

> Coming from Celery? Celery defaults to JSON; taskito's Python default is
> `SmartSerializer` (MessagePack with a cloudpickle fallback). The fallback
> **executes code on load**, and it's Python-only — use `JsonSerializer` for
> cross-language or untrusted payloads.

</SdkOnly>

<SdkOnly sdk="python">

```python
from taskito import Queue, JsonSerializer

# Use JSON for simpler, cross-language payloads
queue = Queue(serializer=JsonSerializer())
```

</SdkOnly>

<SdkOnly sdk="node">

```ts
import { Queue, MsgpackSerializer } from "@byteveda/taskito";

// Use msgpack for compact, binary payloads
new Queue({ dbPath: "taskito.db", serializer: new MsgpackSerializer() });
```

</SdkOnly>

<SdkOnly sdk="java">

```java
import org.byteveda.taskito.Taskito;
import org.byteveda.taskito.serialization.MsgpackSerializer;

// Use msgpack for compact, binary payloads
Taskito queue = Taskito.builder()
    .sqlite("taskito.db")
    .serializer(new MsgpackSerializer())
    .open();
```

</SdkOnly>

## Built-in serializers

<SdkOnly sdk="python">

| Serializer | Format | Best for |
|---|---|---|
| `SmartSerializer` (default) | MessagePack + cloudpickle fallback | Mixed payloads; falls back to pickle for complex objects |
| `CloudpickleSerializer` | Binary (pickle) | Complex Python objects, lambdas, closures |
| `JsonSerializer` | JSON | Simple types, cross-language interop, debugging |

</SdkOnly>

<SdkOnly sdk="node">

| Serializer | Format | Best for |
|---|---|---|
| `JsonSerializer` (default) | JSON | Simple types, cross-language interop, debugging |
| `MsgpackSerializer` | Binary (msgpack) | Compact payloads, smaller storage |
| `SignedSerializer` | Wraps a codec + HMAC-SHA256 | Tamper detection |
| `EncryptedSerializer` | Wraps a codec + AES-256-GCM | Encrypted, authenticated payloads |

</SdkOnly>

<SdkOnly sdk="java">

| Serializer | Format | Best for |
|---|---|---|
| `JsonSerializer` (default) | JSON (Jackson) | Simple types, cross-language interop, debugging |
| `MsgpackSerializer` | Binary (msgpack) | Compact payloads, smaller storage |
| `SignedSerializer` | Wraps a serializer + HMAC-SHA256 | Tamper detection |
| `EncryptedSerializer` | Wraps a serializer + AES-GCM | Encrypted, authenticated payloads |

</SdkOnly>

## Custom serializers

Implement the `Serializer` interface with <SdkSwap python={<><code>dumps(obj)</code> / <code>loads(data)</code></>} node={<><code>serialize(value)</code> / <code>deserialize(bytes)</code></>} java={<><code>serialize(value)</code> / <code>deserialize(bytes, type)</code></>} /> methods.

## What gets serialized

- **Arguments**: serialized and stored as a BLOB in `payload`
- **Results**: serialized and stored as a BLOB in `result`
- **Periodic task args**: serialized at registration time, stored as BLOBs in
  `periodic_tasks.args`
