See [Pub/Sub](/node/guides/core/pubsub) for the full guide (delivery
semantics, lifecycle, cross-SDK topics).

| Method | Description |
|---|---|
| `subscriber(topic, name, fn, options?)` | Register `fn` as `name`, subscribed to `topic`. Returns the queue (chainable, like `task`). |
| `publish(topic, args?, options?)` ā†’ `Promise<Job[]>` | Fan a message out to every active subscription ā€” one job each. Empty array when nothing is subscribed. |
| `declareSubscriptions()` ā†’ `Promise<void>` | Write pending durable subscriptions to storage. Call in a producer-only process; `runWorker()` does this automatically. |
| `unsubscribe(topic, name)` ā†’ `Promise<boolean>` | Remove a subscription. |
| `pauseSubscription(topic, name)` / `resumeSubscription(topic, name)` ā†’ `Promise<boolean>` | Stop/resume deliveries without unregistering. |
| `listSubscriptions(topic?)` ā†’ `Promise<Subscription[]>` | All subscriptions, or one topic's active ones. |
| `listTopics()` ā†’ `Promise<string[]>` | Distinct topics with at least one subscription. |
| `reapEphemeralSubscriptions()` ā†’ `Promise<number>` | Drop ephemeral subscriptions whose owning worker is gone. Runs automatically on the worker heartbeat cadence; exposed for operational tooling. |
| `subscribeLog(topic, name)` ā†’ `Promise<void>` | Register a durable log subscription ā€” a named cursor with no handler. Writes immediately, so register it before the publishes it should see. |
| `logConsumer(topic, name, handler, opts?)` ā†’ `this` | Register a **managed consumer**: the durable log subscription plus, once a worker runs, a poll loop that pulls messages, calls `handler(...args)` per message (the handler may return a `Promise` ā€” it is awaited), and advances the cursor. `opts`: `pollIntervalMs` (default 1000), `batchSize` (default 100), `onError` (`"retry"` leaves a failed message un-acked to re-read, default; `"skip"` acks past it). |
| `readTopic(topic, name, limit?)` ā†’ `Promise<TopicMessage[]>` | Pull up to `limit` (default 100) messages after `name`'s cursor, oldest first and exclusive of it. Empty once caught up. At-least-once ā€” process, then `ackTopic()`. |
| `ackTopic(topic, name, cursor)` ā†’ `Promise<boolean>` | Advance a log subscription's cursor to `cursor` (a message `id`) ā€” a monotonic high-water mark. `false` if nothing moved. |
| `topicLogStats()` ā†’ `Promise<TopicLogStat[]>` | Lag snapshot for every log subscription: `topic`, `subscription`, `cursor`, `lag`, `oldestUnackedAgeMs`. |

See [Log topics](/node/guides/core/pubsub#log-topics) for the cursor,
at-least-once, and retention semantics.

## `TopicMessage`

One message pulled from a log topic, returned by `readTopic()`.

| Field | Type | Description |
|---|---|---|
| `id` | `string` | Cursor token ā€” pass to `ackTopic()`. |
| `args` | `unknown[]` | Deserialized positional args from the `publish()` call. |
| `metadata` | `Record<string, unknown>` | Caller metadata, if any. |
| `notes` | `Record<string, unknown>` | Structured notes, if any. |
| `createdAt` | `number` | Unix-millisecond publish time. |
