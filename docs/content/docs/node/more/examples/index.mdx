---
title: Examples
description: "End-to-end, runnable TypeScript examples — each demonstrating a slice of Taskito's Node SDK in a realistic setting."
---

Each example is a self-contained project you can copy, run, and adapt. They build
on the [core guides](/node/guides/core/tasks) — read those for the concepts, come here for working code.

| Example | Demonstrates |
| --- | --- |
| [Express service](/node/more/examples/express-service) | REST API that enqueues jobs, streams progress over SSE, and cancels in flight |
| [Notification service](/node/more/examples/notifications) | Delayed scheduling, idempotency, priority, periodic tasks, batch enqueue |
| [Web scraper](/node/more/examples/web-scraper) | Rate limiting, retries, named queues, middleware hooks, fan-out workflows |
| [ETL data pipeline](/node/more/examples/data-pipeline) | A multi-stage DAG with progress, error history, and cascade cancellation |
| [DAG workflows](/node/more/examples/workflows) | Fan-out/fan-in, gates, conditions, sub-workflows, caching, critical-path analysis |
| [Saga — checkout](/node/more/examples/saga-checkout) | Compensating transactions with reverse-order rollback |
| [Bulk emails](/node/more/examples/bulk-emails) | High-volume `enqueueMany` with concurrency caps and rate limits |
| [Predicate-gated jobs](/node/more/examples/predicate-gated-jobs) | Enqueue-time gates with `allOf` / `anyOf` / `not` predicate composition |
| [Benchmark](/node/more/examples/benchmark) | Measuring enqueue, processing, and end-to-end latency and throughput |

Every snippet targets the standalone Node SDK — `import { Queue } from "taskito"` —
with no Python in the loop. Pick the one closest to your problem and start there.
