---
title: Predicate-Gated Jobs
description: "Reject enqueues that don't meet a policy at submit time, composing business-hours, feature-flag, and quota predicates."
---

A predicate is a function evaluated when a job is enqueued. If it returns `false`,
the enqueue is rejected with a `PredicateRejectedError` and no job is created —
backpressure at the front door, before any work is staged.

## What this gates

- promotional sends only during business hours,
- an expensive re-index only when its feature flag is on,
- a tenant's exports only while under quota.

## predicates.ts

A predicate receives `{ taskName, args }` — the positional args after any
`onEnqueue` rewrites. Compose them with `allOf` / `anyOf` / `not`.

```ts
import type { Predicate } from "taskito";

export const duringBusinessHours: Predicate = () => {
  const hour = new Date().getUTCHours();
  return hour >= 13 && hour < 22; // 9–6 ET
};

export const featureEnabled = (flag: string): Predicate => () => flags.isOn(flag);

export const underQuota: Predicate = ({ args }) => {
  const [tenantId] = args as [string];
  return usage.exports(tenantId) < usage.limit(tenantId);
};
```

## app.ts

`queue.gate` attaches one or more predicates to a task; every gate must pass.

```ts
import { Queue, allOf, not } from "taskito";
import { duringBusinessHours, featureEnabled, underQuota } from "./predicates";

export const queue = new Queue({ dbPath: "gated.db" });

queue.task("sendPromo", (audience: string) => mailer.blast(audience));
queue.task("reindex", () => search.rebuild());
queue.task("exportData", (tenantId: string) => exporter.run(tenantId));

queue.gate("sendPromo", duringBusinessHours);
queue.gate("reindex", featureEnabled("expensive-reindex"));
queue.gate("exportData", allOf(underQuota, not(duringBusinessHours))); // off-peak only
```

## Enqueuing against a gate

A rejected enqueue throws synchronously — handle it where you submit.

```ts
import { PredicateRejectedError } from "taskito";

try {
  queue.enqueue("sendPromo", ["newsletter-subscribers"]);
} catch (err) {
  if (err instanceof PredicateRejectedError) {
    res.status(409).json({ error: "blocked by policy", task: err.taskName });
  } else {
    throw err;
  }
}
```

## Observing outcomes

Gates run before a job exists, so there is no job event for a rejection — the
thrown error is your signal. Subscribe to the lifecycle events to observe jobs
that *did* pass the gate.

```ts
queue.on("job.completed", (e) => console.log(`${e.taskName} ${e.jobId} ok`));
queue.on("job.dead", (e) => console.warn(`${e.taskName} ${e.jobId} dead: ${e.error}`));
```

<Callout type="info">
  Predicates run at **enqueue time** in the producer process, synchronously.
  Keep them cheap and side-effect-free — a slow predicate slows every enqueue.
  For checks that need the worker (resource state, external I/O), gate inside the
  task and fail it instead.
</Callout>

## Key patterns

| Pattern | Where |
| --- | --- |
| Attach a gate | `queue.gate(task, predicate)` |
| Compose policies | `allOf` / `anyOf` / `not` |
| Args-aware gate | `({ args }) => …` |
| Handle rejection | `catch (PredicateRejectedError)` |
| Observe passers | `queue.on("job.completed" \| "job.dead")` |
