---
title: Job Management
description: "List and inspect jobs, manage the dead-letter queue, and pause queues."
---

Read and manage individual jobs directly from the `Queue` handle. For
queue-wide health — stats, per-task metrics, progress, workers — see
[Monitoring](/node/guides/observability/monitoring).

## Jobs

```ts
await queue.listJobs({ status: "failed", limit: 50 });
await queue.getJobErrors(id); // per-attempt error history
```

## Dead letters

```ts
await queue.deadLetters();
queue.retryDead(deadId);
queue.deleteDead(deadId);
await queue.purgeDead(olderThanMs);
await queue.purgeCompleted(olderThanMs);
```

<Callout type="info">
  Scan-heavy methods (`listJobs`, `getJobErrors`, `deadLetters`, `purgeDead`,
  `purgeCompleted`) run off the JS event loop on a background thread pool and
  return a `Promise`.
</Callout>

## Pause / resume

```ts
queue.pauseQueue("default");
queue.resumeQueue("default");
queue.listPausedQueues();
```

<Callout type="info">
  The same data drives the [dashboard](/node/guides/operations/dashboard) and the
  [CLI](/node/guides/operations/cli) read commands.
</Callout>
