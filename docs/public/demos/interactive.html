Rate limiting

What happens when your provider says 429?

Requests flow your code → queue → workers → external API. Flip the API to Too Many Requests and watch taskito hold the line: failed calls turn red, retry counters climb, an exponential backoff timer kicks in, and workers keep draining everything else. Nothing is dropped.

Failure recovery

Scrub one job through its whole lifecycle.

Drag the playhead across a single job's timeline — queued → running → failed → retrying → running — and watch the live state, attempt counter, and event log update at every step. Then switch the ending: does it recover on the last retry, or exhaust them and land in the dead-letter queue?

Worker scaling

Tune the pool. Watch throughput, depth, and latency react.

A live discrete-event simulation. Move the sliders for workers, concurrency, arrival rate, and a per-second rate limit — the throughput, queue-depth, and latency charts respond in real time. Find the point where capacity meets demand, and the moment the queue runs away from you.

Workflow execution

Fire a DAG and watch execution propagate.

A real dependency graph — chains run in sequence, groups fan out in parallel, a chord joins them back. Hit Run and each task lights up the moment its inputs are ready, or click any node to trace its downstream path and run just that branch.

Mesh scheduling

Route every job to the node built to run it.

Because the queue lives in a shared store, any number of worker nodes pull from it — no broker. Each job's routing key sends it to the right pool (gpu, default, email). Click any machine to drop it and watch its work reroute across the rest of the mesh.

Saga · compensation

When a step fails, unwind the ones that already ran.

A booking modeled as a saga — each step carries a compensating action. Pick where it fails and run it: forward steps commit one by one, then taskito fires the compensations in reverse order so nothing is left half-done. Either it all commits, or it all rolls back.

Work-stealing

Idle regions steal work from the busy ones.

Run taskito in several regions and they gossip queue depth to each other. When one region's backlog spikes, an idle peer pulls a batch over HTTP and runs it — so no region drowns while another sits idle, and there's still no central broker. Hit Burst to overload a region and watch the mesh rebalance.
